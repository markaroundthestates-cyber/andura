# SECTION 09 — i18n completeness (EVERY user-facing string EN+RO, zero hardcode, zero diacritics)

> **Weight 7% · Gate 100% · Critical: no (but gate is 100% — a single FAIL fails the section).**
>
> **Goal.** Every user-facing string the app can render — JSX text, `title`,
> `aria-label`, `placeholder`, `alt`, `aria-description`, toast/error copy,
> button labels, empty-state hints — goes through `t()` / `tArray()`. EN + RO
> both present for every key (`src/i18n/en.json` ↔ `src/i18n/ro.json`, identical
> key structure, 1366 keys each as of 2026-05-29). RO values carry **no
> diacritics** (D-LEGACY-064). Engine outputs emit a *semantic key*
> (`readiness.key`, `fatigue.key`, `coachVoice` pool key) and the React layer
> localizes via `t()`/`tArray()` — **no raw RO/EN engine sentence reaches the
> DOM**. Locale switches live (RO↔EN) from `Cont > Setari > Limba` with no
> remount artifact; default locale `en` (2026-05-28 flip); persisted under
> `localStorage['sf.locale']`.
>
> **Why this section exists / what it must catch.** Daniel's 2026-05-29 smoke
> found `InstallPrompt.tsx` rendering literal Romanian ("Instaleaza Andura" /
> "Acces rapid din ecranul de start." / "Instaleaza" / aria-label "Inchide")
> under EN locale — even though `install.*` keys already existed in both
> bundles. The root cause of the *class* of bug: the existing zero-leak harness
> (`src/i18n/__tests__/i18nNoRoLeak.test.tsx`) is a **manually-curated import
> table** — a component only gets checked if someone remembered to add it.
> InstallPrompt was never imported into that test → it leaked. The same hole
> covers `UpdatePrompt`, `OfflineBanner`, `Toast`, `SessionPill`,
> `LoadingSkeleton`, `MedicalDisclaimerModal`, `AlertsBanner`, `PatternsBanner`,
> `InactivityPrompt`, `NotFound`, `AuthCallback`, `Privacy`, `Terms`. This
> section enumerates **every** `.tsx` under `src/react/routes/screens/**` (48
> files) and `src/react/components/**` (57 files) as individual per-file checks,
> plus the bundle-parity / diacritics / engine-boundary / locale-switch checks.
>
> **The machine bar (HARNESS).** Two test files are the deterministic gate for
> the per-file steps:
> - `src/i18n/__tests__/i18nNoRoLeak.test.tsx` — renders a curated screen list
>   under `setLocale('en')` and asserts zero diacritics + zero forbidden RO
>   token (`FORBIDDEN_RO_TOKENS` ~250 words). **It is incomplete** — the audit's
>   primary deliverable is to extend it to ALL files (see §[09.900]).
> - `src/i18n/__tests__/i18n.test.js` — bundle integrity (RO↔EN key parity, all
>   leaves non-empty, RO no-diacritics, fallback chain, locale persistence).
> - **`src/react/lib/__tests__/noHardcodedUiStrings.test.ts`** — the
>   *static AST/grep scanner* prescribed by 00-MASTER as the "machine bar for the
>   per-file steps". **It does NOT exist yet** (verified 2026-05-29) — building
>   it is step §[09.901]. Until it exists, the per-file `Verify` below uses a
>   reproducible `grep` + a render-under-EN assertion as the manual bar.
>
> **How a per-file step is judged.**
> - `PASS` — file imports `i18n/index` (or is a pure presentational primitive
>   with provably zero user-facing prose) AND grep for hardcoded RO/EN prose in
>   JSX text / `title=` / `aria-label=` / `placeholder=` / `alt=` returns zero,
>   AND it renders EN-clean under `setLocale('en')`.
> - `FAIL` — any hardcoded user-facing string (file:line evidence).
> - `PARTIAL` — mostly wired but one attribute (often `aria-label`/`alt`/`title`)
>   left hardcoded, or RO present but with a diacritic.
> - `BLOCKED` — only if the file genuinely cannot be rendered/inspected.
>
> **Default locale for all render checks is `en`** (the 2026-05-28 flip). A RO
> word appearing under EN is the canonical leak. Diacritics are forbidden under
> BOTH locales.
